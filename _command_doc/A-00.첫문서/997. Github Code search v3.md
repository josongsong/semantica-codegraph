대규모 리포지토리 대응	수십억 라인 규모의 코드베이스를 검색 가능해야 함. 여러 레포지토리를 넘나들어 검색 가능. 
GitHub
+1

언어 인식 & 문맥 인식 검색	단순 키워드 매칭이 아니라 코드의 구조나 심볼을 인식하고, “심볼 이름”, “파일 이름”, “문맥” 등을 고려할 수 있어야 함. 
GitHub
+1

강력한 검색 표현식	정규표현식, Boolean 연산자, 특수 문자 지원 등 고급 쿼리 언어 제공. 예: /limits?/ repo:…처럼 검색식 가능. 
GitHub
+1

빠른 응답 & 최신 상태 유지	코드가 업데이트 되면 인덱스도 빠르게 반영되어야 하고, 검색 결과도 최신 커밋을 반영해야 함. 
GitHub

검색 결과에서 코드 탐색 가능	검색 결과 목록만 주는 게 아니라, 해당 코드로 바로 이동하고 “코드 보기 + 탐색”이 가능해야 함 (“코드 뷰(code view)” 기능) 
GitHub
+1

높은 관련도 랭킹	단순 매칭 수보다 “어떤 결과가 더 관련 있는가”를 판단하는 랭킹 메커니즘이 중요함. 코드 검색임에도 유사도/문맥이 고려됨. 
The GitHub Blog
+1

다양한 필터/쿼리 옵션	언어 필터, 파일 경로 필터, 리포지토리 필터 등이 제공되어야 함 (예: language:ruby OR language:python) 
GitHub

사용자 경험 측면 강화	키보드 단축, 탐색 이동, 파일/심볼 바로가기 등 검색 후 탐색이 쉽게 되어야 함.
