# AI-generated Code & Document Usage Specification

본 문서는 AI가 생성한 코드와 문서 사용에 대한 규칙을 정의함.

---

## 목적

- AI 생성 결과에 대한 책임 범위 명확화
- 품질·보안·일관성 기준 유지
- AI 도입으로 인한 무분별한 규칙 위반 방지

---

## MUST 규칙

1. AI가 생성한 코드/문서에는 커밋 메시지 또는 PR 설명에 **`[ai]` 태그**를 포함해야 함.
2. AI가 생성한 코드는 사람에 의한 리뷰를 반드시 거쳐야 함.
3. AI가 생성한 코드는 기존 lint/format/type-check 규칙을 100% 통과해야 함.
4. 민감 정보(credential, 토큰, 비공개 API 키)를 프롬프트로 제공해서는 안 됨.
5. 외부 레포/코드에서 복사된 것으로 의심되는 결과물은 라이선스 관점에서 검토해야 함.
6. 스펙(specification) 문서를 AI로 생성·수정한 경우, 최종 책임자는 사람으로 명시해야 함.

---

## 금지 규칙 (MUST NOT)

1. 테스트 없이 AI 생성 코드를 바로 배포.
2. 보안 관련 설정/키/시크릿을 AI 대화에 직접 포함.
3. 외부 저작권 코드가 포함된 결과물을 그대로 사용하는 행위.
4. AI가 제안한 아키텍처/설계를 스펙 업데이트 없이 바로 구현하는 행위.
5. AI가 생성한 코드를 “검증 불필요”로 간주하는 태도.

---

## 문서 간 경계

- 보안/시크릿 처리 규칙은 `SECURITY_SPEC.md`.
- 코드 리뷰 기준은 `CODE_REVIEW_SPEC.md`.
- 문서 구조 및 계층 규칙은 `docs/README.md` 에서 별도 정의함.
