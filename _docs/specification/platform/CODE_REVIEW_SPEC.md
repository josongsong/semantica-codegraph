# Code Review Specification

본 문서는 Semantica 레포 전역에서 적용되는 PR 리뷰 규칙을 정의한다.
리뷰 기준은 프로젝트 품질을 유지하는 핵심 계약이며 MUST 수준이다.

---

## 목적

- 리뷰 품질 표준화
- 스펙 업데이트 누락 방지
- 코드/테스트/문서 간 일관성 유지
- 변경 영향 범위를 안정적으로 관리

---

## MUST 규칙

1. 모든 변경(PR)은 아래 항목을 만족해야 함
   - 코드 변경
   - 테스트 변경
   - 문서(specification 또는 standards) 변경
   - 주석/에러/로그 필드 일관성 확인

2. 리뷰어는 아래 관점을 모두 확인해야 함
   - 스펙 준수 여부
   - 레이어 규칙(LAYERING_SPEC.md) 준수 여부
   - 타입/린트/포맷 통과 여부
   - 변경 영향 범위(Upstream/Downstream)
   - 성능 및 리소스 영향
   - 보안 규칙(SERCURITY_SPEC.md) 위반 여부

3. PR은 반드시 다음 내용을 포함해야 함
   - 변경 요약
   - 변경 이유
   - 영향 범위
   - 관련 스펙 및 문서 링크
   - 테스트 결과

4. 하나의 PR은 하나의 의도를 가져야 함.
   - 기능 추가 / 리팩토링 / 문서화 / 버그 수정 분리

---

## 금지 규칙

1. 대규모 변경을 단일 PR로 제출
2. 테스트/문서 업데이트 없는 기능 변경
3. 린트 disable로 문제 무시
4. 레이어 경계 위반을 리뷰에서 허용
5. “approve 없이 merge”
